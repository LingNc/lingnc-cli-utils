## 项目概述

本项目开发了多个小工具，涉及范围较为广泛，使用语言也多种多样。该文档旨在帮助来进行开发的Agent遵守该项目的开发规范、了解项目概况和实时状态。

---

## 快速开始

每个工具都有其自己的使用说明README（也可能没有），也可能会在其各自的工具文件夹中有PLAN(计划文档)和TODO，这些通常指导你完成接下来的开发。并且通常有其自己的 .gitignore 文件。

接下来介绍一下项目的组成: 对于顶层目录，AGENTS.md是该文档，PLAN.md 记录总的进度状况，RULE.md 是Git提交和分支策略。每个文件夹对应一个小工具，开发时均在其文件夹名对应的特性分支中完成开发（详细参考 RULE.md ）。

## 状态速览

该栏目存放当前正在开发的小工具的进度、日志、开发规范、使用说明、文档对应文件名等。该部分应在离开该技能对应开发分支时清除对应的内容。当首次开发该工具时应开始建补充对应的内容，若不需要再保持空。

## Agent 自举说明

**本文档作为项目全局状态机与综述，能够防止代码库膨胀后上下文丢失，确保后续 AI 辅助开发时拥有完整的记忆和设计初衷。**

### 实时同步要求
1. 当 PLAN.md 中的任务完成时应实时更新状态。
2. 当修复关键 Bug 时，必须如实记录。
3. 当添加新功能时，必须记录功能说明和改动。
4. 当完成开发任务时，必须进度清单。
5. 当添加新的技术规范时，必须补充。
6. 当功能有较大变更时，必须同步更新 README.md。
7. **隐私策略**：所有文档中不得出现敏感信息。

### 维护责任
- 每次会话开始前，AI 工具应优先读取 AGENTS.md 了解项目状态
- 每次重要变更后，AI 应及时更新。
- 保持 AGENTS.md 与其余部分的描述一致性。
- **Git 操作规范**：根据 RULE.md 执行 git 操作（详细请参考RULE.md），任何git操作前均首先确保自己熟知 RULE.md 文档内容。

### 必须做的事
- 完成用户的任务后，必须回复：“我已严格遵守AGENTS.md。”。
